서버란 무엇인가

서버는 컴퓨터가 아니다​

서버라고 하면 많은 사람들이 좁은 공간에 빽빽하게 놓인 기계 장치와, 그 장비를 관리하는 숙련된 엔지니어를 떠올린다. 이런 이미지 때문에 서버를 ‘큰 컴퓨터’나 ‘특별한 공간’으로 오해하기 쉽다.

하지만 서버는 컴퓨터도, 공간도 아니다.
서버는 하나의 역할을 가진 프로그램이다.

물리적인 컴퓨터 위에서 실행될 수는 있지만, 그 컴퓨터 자체가 서버인 것은 아니다.

서버는 항상 실행 중인 프로그램이다​

그렇다면 서버는 어떤 프로그램일까?

서버의 역할은 단순하다.
요청이 들어오면, 그 요청에 맞는 응답을 돌려주는 것이다. 이를 위해 서버는 항상 실행된 상태로 요청이 도착하기를 기다린다.

게임 서버는 사용자의 행동 요청을 기다리고, 이메일 서버는 메일 수신 요청을 기다린다. 서버의 종류는 다르지만, “요청을 기다린다”는 점은 모두 동일하다.

서버는 무엇을 기다리고 있는가​

서버가 기다리는 것은 막연한 접속이 아니라, **형식이 정해진 요청(Request)**이다.

각 클라이언트는 필요한 작업을 수행하기 위해 약속된 형식으로 요청을 보내고, 서버는 이를 해석해 적절한 응답(Response)을 생성한다.

서버는 스스로 무언가를 시작하지 않는다. 항상 클라이언트의 요청이 출발점이다.

동시에 여러 클라이언트를 상대한다는 것​

아 클라이언트의 요청을 처리하는것이 서버구나. 그럼 클라이언트의 수 만큼 서버가 존재해야 하는건가...?

결론은 아니다. 클라이언트의 입장에서 서버는 하나처럼 보인다. 하지만 서버의 입장에서 상황은 다르다.

서버는 동시에 수많은 클라이언트의 요청을 처리해야 한다. 예를 들어, 메이플스토리와 같은 게임 서버는 하루에 수만에서 수십만 명의 사용자가 접속한다.

이 때문에 서버는 “한 명씩 처리하는 프로그램”이 아니라, 여러 요청을 동시에 다룰 수 있도록 설계된 프로그램이어야 한다.

서버가 멈추면 어떤 일이 벌어질까​

서버가 실행 중이 아니라면, 요청을 받아줄 대상이 존재하지 않는다.

그 결과 클라이언트는 응답을 받을 수 없고, 사용자는 아무 동작도 수행할 수 없다.

서버가 “항상 실행 중”이어야 하는 이유는 바로 여기에 있다.

이 문서에서 다루지 않는 것​

이 문서에서는 다음 내용을 다루지 않는다.

  • 포트와 소켓
  • 요청을 어떻게 전달하는지
  • 여러 요청을 동시에 처리하는 구체적인 방법

이제 서버가 무엇인지 이해했다면, 다음 문서에서는 서버가 외부와 어떻게 연결되는지를 살펴본다.
