# 웹이란 무엇인가

## 우리가 매일 사용하는 웹
WWW(World Wide Web)는 멀리 떨어진 컴퓨터들이 서로 통신할 수 있도록
연결된 환경을 의미한다.  
하나의 PC가 다른 나라에 있는 서버와 데이터를 주고받을 수 있게 해주는
통신의 구조가 바로 웹이다.

예를 들어, 프랑스에 있는 내가 한국 판교에 위치한
메이플스토리 서버가 제공하는 게임을 문제없이 즐길 수 있는 것도
이러한 웹 환경 덕분이다.
물리적으로는 멀리 떨어져 있지만,
네트워크를 통해 논리적으로는 연결되어 있다.

## 웹은 페이지가 아니라 통신이다
그렇다면 프랑스에 있는 내가
어떻게 한국에 있는 메이플스토리 서버와 상호작용할 수 있을까?
웹은 이 과정에서 어떤 역할을 하고 있을까?

메이플스토리 플레이 과정을 아주 단순화하면 다음과 같다.

- 게임을 설치하고 실행한다  
  → 클라이언트 프로그램을 실행한다
- 몬스터 한 마리를 잡는다  
  → 클라이언트가 서버로 요청(Request)을 보낸다
- 경험치를 얻는다  
  → 서버가 처리 결과를 응답(Response)으로 돌려준다
- 게임을 종료한다  
  → 클라이언트 프로그램을 종료한다

이 과정에서 중요한 점은,
사용자는 “게임 화면”만 보고 있지만
그 이면에서는 **요청과 응답이라는 통신이 계속 반복되고 있다는 것**이다.
웹은 페이지 자체가 아니라,
이러한 통신이 이루어지도록 하는 기반이다.

## 클라이언트와 서버
웹 통신에는 항상 두 역할이 등장한다.

- **클라이언트(Client)**  
  요청을 보내는 쪽이다.  
  게임 클라이언트, 웹 브라우저, 모바일 앱 등이 이에 해당한다.

- **서버(Server)**  
  요청을 받아 처리하고, 그 결과를 응답으로 돌려주는 쪽이다.  
  서버는 항상 실행 중이며,
  여러 클라이언트의 요청을 동시에 처리해야 한다.

웹은 이 두 역할이 명확히 분리된 구조 위에서 동작한다.

## 요청(Request)과 응답(Response)
클라이언트와 서버는 아무 말이나 주고받지 않는다.
서로 약속된 형식에 따라 통신한다.

클라이언트는
“이 행동을 처리해 달라”는 요청(Request)을 보내고,
서버는
“처리 결과는 이것이다”라는 응답(Response)을 보낸다.

웹에서 일어나는 모든 일은
이 요청과 응답의 반복으로 설명할 수 있다.

## 브라우저는 무엇을 하고 있는가
웹 브라우저는 서버가 아니다.
브라우저는 **웹 통신을 수행하는 클라이언트 프로그램**이다.

주소를 입력하거나 버튼을 클릭하면,
브라우저는 서버로 요청을 보내고,
서버가 돌려준 응답을 해석해
사용자가 볼 수 있는 화면으로 보여준다.

즉, 브라우저는
“웹 페이지를 가지고 있는 존재”가 아니라
“서버와 대화하는 역할”을 맡고 있다.

## 이 문서에서 다루지 않는 것
이 문서에서는 다음 내용을 다루지 않는다.

- HTML, CSS, JavaScript
- HTTP 메서드와 상태 코드
- 실제 서버 구현 방식

이 문서의 목적은
웹을 구성하는 최소한의 개념을 정리하는 것이다.
이 개념을 바탕으로,
다음 문서에서는 서버가 무엇인지 살펴본다.
